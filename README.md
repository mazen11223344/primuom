# Primuom

نظام استثمار متكامل لإدارة الحسابات الاستثمارية، الإيداعات، السحوبات، الأرباح اليومية، والدعم الفني.

## الإعداد المحلي

```bash
npm install
npm run dev
```

## الإعداد على Vercel

من أجل حفظ البيانات بشكل دائم على Vercel، يستخدم المشروع خدمة **Vercel KV** (Upstash Redis).  
أضف المتغيرات التالية في صفحة **Environment Variables** في إعدادات مشروع Vercel (وأيضاً في ملف `.env.local` عند التطوير محلياً):

| المتغير | الوصف |
| --- | --- |
| `KV_REST_API_URL` | الرابط الذي توفره Vercel KV |
| `KV_REST_API_TOKEN` | مفتاح الوصول للقراءة/الكتابة |

> يمكن الحصول على القيم عبر: Vercel Dashboard → Storage → Add Integration → Vercel KV → View Details.

في حال لم يتم ضبط هذه المتغيرات، سيعود التطبيق تلقائياً إلى تخزين البيانات في ملفات JSON داخل مجلد `data` أثناء التطوير المحلي فقط.

## بنية البيانات

يتم حفظ البيانات التالية داخل Vercel KV:

- `users` ‎→ جميع بيانات المستخدمين
- `passwords` ‎→ كلمات مرور المستخدمين (نص عادي حالياً، يوصى لاحقاً بالتشفير/الهاش)
- `networks` ‎→ شبكات الإيداع
- `withdrawals` ‎→ طلبات السحب
- `support_tickets` ‎→ رسائل الدعم الفني

## أوامر مفيدة

```bash
# تشغيل التطوير
npm run dev

# بناء النسخة النهائية
npm run build

# تشغيل النسخة النهائية محلياً
npm run start
```
